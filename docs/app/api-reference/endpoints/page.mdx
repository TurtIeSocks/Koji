## Calculation (`/calc`)

<Table>
  <thead>
    <Table.Tr>
      <Table.Th>Method</Table.Th>
      <Table.Th>Path</Table.Th>
      <Table.Th>Description</Table.Th>
    </Table.Tr>
  </thead>
  <tbody>
    <Table.Tr>
      <Table.Td>`POST`</Table.Td>
      <Table.Td>`/calc/bootstrap`</Table.Td>
      <Table.Td>Builds bootstrap points from `area`, `instance`, or `parent`, optionally saves to DB/scanner.</Table.Td>
    </Table.Tr>
    <Table.Tr>
      <Table.Td>`POST`</Table.Td>
      <Table.Td>`/calc/{mode}/{category}`</Table.Td>
      <Table.Td>Clusters and routes points for a category. Typical `mode`: `cluster` or `route`.</Table.Td>
    </Table.Tr>
    <Table.Tr>
      <Table.Td>`POST`</Table.Td>
      <Table.Td>`/calc/reroute`</Table.Td>
      <Table.Td>Re-routes existing clusters/data points without re-clustering.</Table.Td>
    </Table.Tr>
    <Table.Tr>
      <Table.Td>`POST`</Table.Td>
      <Table.Td>`/calc/route-stats`</Table.Td>
      <Table.Td>Computes distance/coverage stats for provided route clusters.</Table.Td>
    </Table.Tr>
    <Table.Tr>
      <Table.Td>`POST`</Table.Td>
      <Table.Td>`/calc/route-stats/{category}`</Table.Td>
      <Table.Td>Computes stats; if `data_points` omitted, loads points from area by category.</Table.Td>
    </Table.Tr>
    <Table.Tr>
      <Table.Td>`POST`</Table.Td>
      <Table.Td>`/calc/area`</Table.Td>
      <Table.Td>Computes geodesic area for polygon/multipolygon features.</Table.Td>
    </Table.Tr>
  </tbody>
</Table>

`category` accepted values (validated by server):

- `gym`
- `pokestop`
- `spawnpoint`
- `fort` (gym + pokestop)
